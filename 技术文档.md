# 方块消消乐 技术文档

## 1. 项目概述

这是一个基于 HTML5 + CSS3 + JavaScript 的网页小游戏。玩家控制一个从画布顶部随机下落的方块，通过左右移动、切换颜色、加速下落等操作，使相邻的相同颜色方块达到 3 个或以上时消除得分。

**核心玩法：**
- 方块从画布顶部中央开始下落在画布底部堆叠
- 每个方块有 3 种可选颜色（从 9 种固定颜色中随机抽取），玩家可以循环切换
- 消除逻辑：相邻（上下左右）相同颜色的方块 ≥ 3 个时自动消除
- 消除后方块上方的方块会自动下落填充空隙，并继续检测消除
- 游戏速度随得分增加而加快
- 当新方块无法放置时游戏结束

## 4. CSS 样式设计（style.css）

### 4.1 设计原则

- **响应式中心布局**：画布使用 `flex` 居中，最小高度 100vh
- **深色渐变主题**：`#1a1a2e → #16213e → #0f3460` 的 135° 渐变

- **一致的按钮风格**：紫色渐变 `#667eea → #764ba2`，悬停有上浮动画

### 4.3 画布尺寸规格

```
画布: 宽度 = w（可设置）, 高度 = w × 9/6 = 1.5w
方块: 边长 = w/3
列数: 3 列（因为 3 × (w/3) = w）
行数: 最多约 4-5 个方块（因为 1.5w / (w/3) ≈ 4.5）
```

**示例：**
- 当宽度 600px 时，高度 = 900px，方块大小 = 200px
- 当宽度 1000px 时，高度 = 1500px，方块大小 ≈ 333px

### 5.4 方块创建与颜色系统

**新方块创建流程（createNewBlock）：**

```javascript
// 1. 移除上一个当前方块
// 2. 从 0-8 中随机打乱，取前 3 个作为当前方块的候选颜色
const shuffled = [...Array(9).keys()].sort(() => Math.random() - 0.5);
currentBlockColors = shuffled.slice(0, 3);  // 例如 [2, 5, 7]
currentColorIndex = 0;                       // 选中第 1 个

// 3. 设置位置：中央列，略高于顶部
currentX = Math.floor(cols/2) * blockSize;   // 列 1（中间列）
currentY = -blockSize;                       // 在画布上方，逐步进入
// 注：3 列的 X 坐标分别为：0, blockSize, 2*blockSize

// 4. 根据 difficulty 决定是否显示数字
// 5. updateCurrentBlock() 更新样式和位置
// 6. 检查是否游戏结束（见 5.7 节）
```

**颜色切换逻辑（changeColor）：**
- `currentColorIndex = (currentColorIndex + 1) % 3`
- 取 `currentBlockColors[currentColorIndex]` 作为实际颜色索引
- 实际颜色索引用于访问 `COLORS` 或 `blockImages`

### 5.5 自定义图片系统

**图片存储格式：** 使用 `FileReader.readAsDataURL()` 将图片转换为 base64 字符串（DataURL）

```
格式示例：data:image/png;base64,iVBORw0KGgoAAA...
```

**优势：**
- 完全在浏览器端处理，无需服务器
- 即时预览
- 重新开始/切换时无需重新上传（数据保留在内存中）

**图片上传流程：**
1. 用户选择多个文件
2. 按文件名字排序（让用户可以通过命名来控制顺序）
3. 读取最多 9 个文件转换为 DataURL
4. 写入 `blockImages` 数组对应位置
5. 重新生成颜色预览块（有图片的显示图片，无图片的显示颜色）

**方块显示优先级：**
```
if blockImages[colorIdx] 存在 → 显示图片（background-image）
else → 显示纯色（background-color: COLORS[colorIdx]）
```

### 5.6 键盘操作

| 按键 | 功能 | 实现 |
|-----|------|------|
| A / a | 左移一列 | `currentX -= blockSize`，边界检查 `newX >= 0` |
| D / d | 右移一列 | `currentX += blockSize`，边界检查 `newX + blockSize <= gameAreaWidth` |
| S / s | 加速下移一格 | `currentY += blockSize`，立即检测碰底/碰撞 |
| Space | 切换颜色 | `currentColorIndex = (currentColorIndex + 1) % 3` |

**设计说明：** 使用字母键（A/D/S）而非方向键，避免与浏览器滚动/焦点行为冲突。

### 5.7 下落与碰撞系统

**下落定时器（startDrop）：**
- 每 **50ms** 执行一次
- `currentY += dropSpeed`
- 检测两种碰撞情况：

1. **碰到底部**：`currentY + blockSize >= gameAreaHeight`
   - 将方块对齐到底部：`currentY = gameAreaHeight - blockSize`
   - 调用 `placeBlock()` 固定

2. **碰到其他方块**：`getCollisionBlock(currentX, currentY)` 返回非空
   - 将方块对齐到碰撞方块的顶部：`currentY = collisionBlock.y - blockSize`
   - 调用 `placeBlock()` 固定

**碰撞检测算法（getCollisionBlock）：**
```javascript
// 返回与 (x, y) 重叠的已放置方块
// 重叠判定：x 坐标相同（同一列），且 y 范围相交
return placedBlocks.find(block => {
    return x === block.x               // 同一列
        && y + blockSize > block.y     // 当前方块底部在目标方块顶部之下
        && y < block.y + blockSize;    // 当前方块顶部在目标方块底部之上
});
```

### 5.8 消除系统（核心算法）

**广度优先搜索（BFS）算法用于相连区域检测：**

```
算法：findConnectedBlocks(startBlock)
输入：起始方块 startBlock
输出：与起始方块同颜色且相连的所有方块数组

1. 初始化：
   - connected = []               // 结果数组
   - visited = new Set()          // 已访问集合，避免循环
   - queue = [startBlock]         // BFS 队列
   - visited.add(startBlock)

2. 当 queue 不为空时：
   a. current = queue.shift()     // 取出队首
   b. 将 current 加入 connected

   c. 获取 current 的 4 个邻居（上下左右）：getNeighbors(current)
   d. 对每个邻居：
      - 如果未访问 且 颜色与 startBlock 相同
         → 标记为已访问
         → 加入 queue

3. 返回 connected
```

**邻居获取（getNeighbors）：**
- 检查上下左右四个方向
- 方向向量：`[(-blockSize, 0), (blockSize, 0), (0, -blockSize), (0, blockSize)]`
- 在 `placedBlocks` 中查找是否存在精确坐标匹配的方块

**消除流程（checkMatches）：**
```
1. 对每个已放置的方块执行 BFS，找到相连同色方块
2. 如果相连方块数 ≥ 3，将这些方块加入 matches 集合
   （使用 Set 去重，避免同一个方块被多次计入）
3. 如果 matches 非空：
   a. removeBlocks(matches) → 动画移除方块并更新数组
   b. score += matches.size * 10 → 每个方块 10 分
   c. updateScore() → 显示新分数，调整速度
   d. 300ms 后执行 dropFloatingBlocks()
```

### 5.9 悬空方块下落系统

**算法：dropFloatingBlocks()**

```
循环（最多 100 次，防止无限循环）：
  1. 将 placedBlocks 按 y 从大到小排序（先处理最底部的方块）
  2. 对每个方块：
     - 如果下方没有方块 且 未到达底部：
        → y += blockSize（下移一格）
        → 标记本轮有方块移动
  3. 如果本轮没有任何方块移动 → 退出循环
  4. 200ms 后再次调用 checkMatches()（连锁消除）
```

**关键：** 先处理底部方块，确保上方方块下落后底部已稳定。

### 5.10 游戏结束检测与重启

**结束条件（checkGameOver）：**
```javascript
// 新创建的方块位置与已放置方块重叠 → 游戏结束
// 或者：已有方块且新方块 Y 坐标 ≥ 顶部方块 Y 坐标
const isOverlapping = placedBlocks.some(block => {
    return block.x === currentX &&
           currentY + blockSize > block.y &&
           currentY < block.y + blockSize;
});

if (isOverlapping || (placedBlocks.length > 0 && currentY >= placedBlocks[0].y)) {
    endGame();
}
```

**endGame() 流程：**
1. `gameRunning = false`，`clearInterval(dropInterval)` 停止下落
2. 创建 `.game-over` DOM 元素，包含：
   - 标题"游戏结束!"
   - 最终得分
   - **完整的设置面板**（同开始屏幕）：画布宽度（滑块+输入框）、难度、图片上传、背景上传
   - "🔄 重新开始"按钮（class="restart-btn"）
3. 为结束面板中的所有控件绑定事件监听器
4. 重新开始按钮点击 → 读取选中的难度 → `restartGame()`

**restartGame() 流程：**
1. `gameArea.innerHTML = ''` 清空画布
2. 重新创建速度/得分显示
3. 重置：`placedBlocks = []`, `score = 0`, `gameRunning = true`, `dropSpeed = 0.5`
4. 创建第一个方块并启动下落

### 5.11 速度系统

**速度计算公式（updateScore）：**
```javascript
const speedLevel = Math.floor(score / 120);     // 每 120 分为一级
dropSpeed = Math.min(0.5 + speedLevel * 0.2, 8);  // 初始 0.5，每次 +0.2，最大 8

// 示例：
//   0-119   分 → 0.5
//   120-239     → 0.7
//   240-359     → 0.9
//   360-479     → 1.1
//   480-599     → 1.3
//   ...
//   达到 4500 分 → 8.0（封顶）
```

**速度单位**：px/帧 × 20 帧/秒（因为 50ms = 20 次/秒）
- 速度 0.5 → 10 px/秒
- 速度 0.7 → 14 px/秒
- 速度 1.0 → 20 px/秒
- 速度 8.0 → 160 px/秒

### 5.12 文件上传处理

**FileReader API 使用：**
```javascript
const reader = new FileReader();
reader.onload = (evt) => {
    // evt.target.result 包含 DataURL 字符串
    // 使用该字符串设置元素的 background-image 或 src
};
reader.readAsDataURL(file);  // 异步操作
```

**批量上传处理策略：**
- 使用 `loaded` 计数器，所有文件读取完成后统一更新 `blockImages`
- 少于 9 张时，剩余位置保持原样（不会被覆盖）
- 文件名排序使用 `localeCompare('zh-Hans-CN')` 支持中文文件名排序

### 6.1 状态变量总结

| 变量 | 类型 | 变化时机 | 重置时机 |
|-----|------|---------|---------|
| `gameAreaWidth` | 数字 | 滑块/输入框变化 | 否（保留用户设置） |

| `currentBlockColors` | 数组 | createNewBlock | 每次创建新方块 |
| `currentColorIndex` | 数字 | 空格键 | 每次创建新方块 → 0 |
| `placedBlocks` | 数组 | placeBlock/removeBlocks | startGame/restartGame → [] |
| `blockImages` | 数组 | 文件上传/清除按钮 | 否（跨游戏保留） |
| `backgroundImage` | 字符串/ null | 文件上传/清除 | 否（跨游戏保留） |
| `difficulty` | 字符串 | 开始/结束时用户选择 | 否（保留选择） |

### 6.2 游戏主循环

```
开始
  ↓
createNewBlock() → 随机3色 → 定位中央顶部
  ↓
startDrop() → setInterval 每 50ms
  ↓
[每一帧]
  ├── currentY += dropSpeed
  ├── 是否碰到底部？→ placeBlock()
  ├── 是否碰到其他方块？→ placeBlock()
  └── 否则 → 更新样式显示新位置
  ↓
placeBlock()
  ├── 创建放置方块 DOM（加入 placedBlocks）
  ├── 100ms 后 checkMatches()
  │    ├── BFS 找相连同色方块 ≥ 3
  │    ├── removeBlocks() → 动画消除
  │    └── 300ms 后 dropFloatingBlocks()
  │         ├── 悬空方块下落
  │         └── 200ms 后再次 checkMatches()（连锁）
  └── createNewBlock() → 创建下一个方块
  ↓
循环直到 checkGameOver() 判定结束
```

## 7. 视觉与 UX 设计要点

### 7.1 画布尺寸动态调整

- **滑块 + 数字输入框双向同步**：拖动滑块 → 数字更新，输入数字 → 滑块移动
- **范围验证**：300 ≤ width ≤ 1000，超出自动限制
- **即时预览**：调整时画布实时变化，帮助用户选择合适大小
- **在开始屏幕和结束屏幕都可设置**：提升体验连续性

### 7.2 颜色/图片系统

- **固定 9 种位置**：每个颜色有固定编号（1-9），便于调试和用户记忆
- **图片优先级高于颜色**：上传图片后，对应编号位置显示图片
- **菜鸟模式数字提示**：方块中心显示编号，帮助新手理解对应关系
- **老鸟模式挑战**：不显示数字，纯靠视觉判断，增加挑战

### 7.3 UI 元素

- **速度/得分显示**：位于画布角落，半透明背景，白色文字带阴影，确保在任何背景下可见
- **开始/结束面板**：居中显示，玻璃拟态风格，与游戏视觉统一
- **按钮层次**：主操作（开始/重新开始）→ 大紫按钮；辅助操作（清除）→ 小红色按钮

## 9. 已知限制与改进建议

### 9.1 当前限制

1. **列数固定为 3**：硬编码 cols = 3，如需更多列需修改多个位置
2. **图片存储在内存**：刷新页面后图片数据丢失，且大图片可能占用较多内存
3. **下落速度上限 10.0**：游戏难度最终趋于稳定
4. **无最高分记录**：刷新/关闭后得分不保存
5. **移动端支持有限**：键盘操作不适合触屏，无触摸手势

### 9.2 可能的改进方向

1. **增加 localStorage 持久化**：保存最高分、图片设置
2. **引入 Web Audio API**：添加消除音效、背景音乐
3. **添加特殊方块**：炸弹（清除周围方块）、彩虹块（匹配任意颜色）
4. **优化图像存储**：压缩图片或限制图片大小
5. **移动端适配**：添加触摸手势（左滑/右滑/下滑/点击切换）
6. **关卡系统**：不同关卡有不同目标、时间限制、障碍物
7. **消除动画增强**：增加粒子效果、得分飘字动画
8. **方块数量可选**：让用户选择 3 列/5 列/7 列等模式
